# Product Links and Deep Links

## Purpose

This document defines how Taja should generate, share, and resolve product links.

It supports a simple flow where:

- a seller shares a product link
- the customer sends that link back in WhatsApp
- the bot resolves the link and opens the correct product context
- AI-generated product suggestions can also include the same canonical link

## Canonical Product URL

| Item | Recommendation |
|---|---|
| Base domain | `https://www.use-taja.com` |
| Product path | `/p/{product-slug}` |
| Example | `https://www.use-taja.com/p/samsung-galaxy-a15` |

## Optional Fallback Format

If the product slug is not enough to guarantee uniqueness, the link can include a short identifier.

| Item | Recommendation |
|---|---|
| Fallback path | `/p/{product-slug}-{short-id}` |
| Example | `https://www.use-taja.com/p/samsung-galaxy-a15-a91k2` |

## Link Resolution Rules

| Input | System Behavior |
|---|---|
| Full canonical product link | Open the exact product |
| Product link with slug only | Resolve by slug and open product |
| Product link with slug plus short id | Resolve exact product and avoid collisions |
| Base `/p/` without product identifier | Show search prompt or recent products |
| Broken or unknown link | Show not found and suggest search |

## WhatsApp Bot Behavior

| Event | Bot Response |
|---|---|
| User sends a valid product link | Bot opens the product details flow |
| User sends a product link from a seller | Bot identifies the item and shows price, stock, and action buttons |
| User sends a product link generated by AI | Bot uses the same link resolver and continues the shopping journey |
| User sends only `/p/` or an incomplete link | Bot asks for the product name or shows search suggestions |
| User sends an invalid link | Bot offers search, browse, or help actions |

## Sharing Rules

| Rule | Requirement |
|---|---|
| One canonical URL | Every product should have one primary public link |
| Stable slug | Slug should remain stable when possible |
| Short ID | Use a short ID if uniqueness is needed |
| Share buttons | Product pages should include copy/share actions |
| WhatsApp friendly | Links should be short enough to paste and resend easily |

## AI Search and Product Links

| Situation | Expected Behavior |
|---|---|
| Customer asks the AI for a product | AI can suggest a product and include the canonical link |
| Customer shares the product link | Bot should trust the link and jump to that product |
| Customer sends a product screenshot | AI can identify the product and reply with the link if confidence is high |
| Customer only knows partial details | Bot can search by text, image, video, or voice and then return the product link |

## Product Page Requirements

| Requirement | Description |
|---|---|
| Product identity | Product page must expose a stable product ID or slug |
| Shareability | Product page should be easy to copy and send |
| Preview data | Link preview should show title, image, and price where supported |
| WhatsApp routing | The backend should be able to map the link back to product context |

## Recommended Backend Logic

| Step | Action |
|---|---|
| 1 | Detect the incoming URL in chat text |
| 2 | Parse the slug or short ID |
| 3 | Look up the product record |
| 4 | If found, load product context |
| 5 | If not found, offer search or browse fallback |

## Example Link Flow

| Step | Example |
|---|---|
| Seller shares product | `https://www.use-taja.com/p/adidas-running-shoe` |
| Customer sends link in WhatsApp | Bot detects the URL |
| Backend resolves product | Product details open in chat |
| Customer taps buy | Cart and checkout start from the same product |

## Error Handling

| Case | Response |
|---|---|
| Deleted product | Show unavailable and suggest alternatives |
| Changed slug | Redirect to the current canonical product URL |
| Duplicate match | Use short ID or direct product ID to resolve |
| Link from old campaign | Redirect or map to the current product record |

## Summary

Taja should treat product links as first-class entry points into the commerce experience.

That means every product can be shared, pasted into WhatsApp, resolved by the bot, and continued from the exact product context without forcing the customer to start over.
