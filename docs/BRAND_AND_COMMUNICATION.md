# Brand and Communication

## Purpose

This document defines how Taja should sound and present itself across product, support, and marketing channels.

## Brand Traits

| Trait | Meaning |
|---|---|
| Clear | Easy to understand |
| Trusted | Feels reliable and safe |
| Helpful | Supports users without friction |
| Modern | Feels current and product-led |
| Human | Speaks like a real team, not a machine |

## Tone Guidelines

| Situation | Tone |
|---|---|
| Product updates | Confident and concise |
| Support replies | Calm and helpful |
| Marketing posts | Clear and energetic |
| Admin communication | Precise and professional |
| Error messages | Simple, direct, and non-blaming |

## Communication Rules

| Rule | Reason |
|---|---|
| Use plain language | Improves understanding |
| Be consistent | Builds trust |
| Avoid unnecessary jargon | Keeps the product accessible |
| Explain next steps | Reduces confusion |
| Keep branding aligned | Makes the product feel unified |

## Brand Use Across Channels

| Channel | How It Should Feel |
|---|---|
| WhatsApp | Friendly, fast, guided |
| Web | Clear, polished, structured |
| Social | Active, thoughtful, credible |
| Support | Calm, respectful, effective |

## Summary

Taja should sound like a product that helps people get things done quickly, clearly, and safely.
