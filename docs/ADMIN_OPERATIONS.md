# Admin Operations

## Purpose

This document defines how internal teams should operate Taja day to day.

## Primary Admin Responsibilities

| Area | Admin Responsibility |
|---|---|
| Users | Review accounts, flags, and restrictions |
| Sellers | Approve, verify, suspend, or support sellers |
| Products | Approve, reject, edit, or remove listings |
| Orders | Review order status, exceptions, and delivery issues |
| Payments | Monitor payment success, refunds, and settlement |
| Payouts | Review withdrawals and seller balances |
| Support | Handle escalations and issue resolution |
| Broadcasts | Send platform-wide announcements |

## Admin Tools

| Tool | Purpose |
|---|---|
| Search and filters | Find records fast |
| Status labels | Show approval, pending, failed, or complete states |
| Audit log | Record who changed what and when |
| Escalation queue | Surface urgent issues |
| Analytics view | Track operations and growth |

## Moderation Rules

| Case | Default Action |
|---|---|
| Fake or misleading product | Reject or remove |
| Suspicious seller behavior | Flag for review or suspend |
| Customer abuse | Warn, restrict, or block |
| Payment mismatch | Hold and investigate |
| Delivery dispute | Escalate to support or logistics |

## Operational Workflows

| Workflow | Expected Admin Action |
|---|---|
| Seller approval | Review documents and activate account |
| Product moderation | Review listing and change status |
| Refund handling | Verify order and payment details |
| Broadcast message | Draft, approve, and send announcement |
| Account deletion review | Confirm policy and process request |

## Permission Model

| Role | Access Level |
|---|---|
| Super admin | Full access |
| Operations admin | Orders, users, support, reports |
| Support agent | Support and user lookups |
| Moderator | Products and seller review |
| Analyst | Read-only reports and metrics |

## Summary

Admin operations should keep the marketplace safe, responsive, and well controlled without slowing down the customer experience.
