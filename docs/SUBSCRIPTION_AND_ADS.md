# Subscription and Ads

## Purpose

This document defines how Taja can make money from optional seller promotions and subscription-like services beyond core commission.

## Monetization Types

| Type | Description |
|---|---|
| Sponsored products | Sellers pay to boost visibility |
| Store subscription | Sellers pay for premium tools or features |
| Content creation | Optional service for product listings or media |
| Featured placement | Seller pays to appear in featured areas |
| Campaign bundles | Temporary promotion package for events or holidays |

## Sponsored Product Model

| Item | Meaning |
|---|---|
| Bid | Amount a seller is willing to pay for promotion |
| Budget | Total amount allocated to a campaign |
| Duration | How long the promotion runs |
| Visibility goal | Search or category exposure |
| Performance metric | Clicks, views, or conversions |

## Subscription Model

| Tier | Example Features |
|---|---|
| Basic | Core seller tools and standard analytics |
| Plus | Better analytics, more visibility, and support tools |
| Pro | Advanced insights, premium placement, or automation |

## Launch Strategy

| Area | Recommendation |
|---|---|
| Pricing | Start simple and easy to understand |
| Access | Keep promotions optional, not forced |
| Billing | Show each charge clearly in statements |
| VAT | Apply tax where required |
| Reporting | Track ad spend separately from commission |

## Operational Rules

| Rule | Requirement |
|---|---|
| Transparency | Sellers should know what they are paying for |
| Separation | Ads and subscriptions should not be mixed into commission |
| Performance tracking | Show sellers what the spend achieved |
| Easy cancellation | Subscriptions should have clear renewal rules |
| Audit logs | All purchases should be recorded |

## Summary

Sponsored products and subscriptions should increase seller value and platform revenue without making the marketplace confusing.
